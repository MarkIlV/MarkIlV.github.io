一、SQL注入

SQL注入是攻击者将恶意的SQL语句插入到应用的输入参数中，再在SQL服务器后台执行的攻击

SQL注入产生的原因：服务器在接受应用的输入参数时，并未做任何过滤，而直接将参数插入到了后端的查询代码中，从而导致了参数中的语句可以在后端被执行

3.盲注

SQL注入的一种，与基于错误的注入不同，盲注点隐藏了我们想看到的错误信息，我们只能通过注入后有无回显两种状态来爆破想要得到的信息

SQL盲注分为两种，一种是布尔盲注，另一种是时间盲注

（0）盲注中常用到的函数

Function Description
Length() 返回字符串的长度
Substr() 截取字符串
Ascii() 返回字符的Ascii编码
Sleep(n) 挂起n秒
if(expr1,expr2,expr3) 如果expr1正确那么执行expr2否则执行expr3
database() 返回当前数据库的名称
version() 返回当前MySQL数据库的版本信息
